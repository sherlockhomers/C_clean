# CleanC（C盘清理助手）— 产品需求文档 v3.1

## 一、产品概述

| 项目 | 说明 |
|------|------|
| 产品名称 | CleanC |
| 产品定位 | 面向普通用户的 Windows C盘深度清理与空间管理桌面工具，内置 AI 智能助手 |
| 技术栈 | Flutter Desktop (Windows) + AI 大模型接入 |
| 目标用户 | C盘空间不足、电脑卡顿的 Windows 普通用户（非技术背景） |

## 二、核心设计原则

1. **安全第一** — 任何删除操作可逆、可跳过，绝不静默删除用户文件
2. **自动化优先** — 能自动完成的绝不手动，一键搞定迁移和路径修改
3. **清晰透明** — 用户始终知道"做了什么、为什么做、结果如何"
4. **渐进式清理** — 从安全到深度分等级推荐，用户自主选择
5. **迁移优于删除** — 能迁移的不删除，帮用户把占空间的内容搬到其他盘
6. **AI 赋能** — 让 AI 帮用户看懂磁盘、做决策、自动化执行，降低使用门槛到零

## 四、功能模块详解

### 模块1：仪表盘（首页）

**目标：让用户一眼看清C盘状况 + AI 辅助决策 + 一键开始**

#### 1.1 磁盘概览卡片（`DashboardCard` Widget）
- C盘使用率环形图（`CircularProgressIndicator`，分段着色：已用/可用/系统占用/用户占用）
- D/E/F 盘可用空间横向条（`LinearProgressIndicator`，提示可迁移目标）
- 实时数据：总容量 / 已用 / 可用 / 可释放预估

#### 1.2 AI 精要摘要（`AISummaryCard` Widget）— 新增核心组件

> **AI 在首页用一句话告诉你：你的C盘怎么了、最该做什么**

```
+-- AI 精要分析 ---------------------------------------------------+
|                                                                    |
|  [AI Icon] 你的C盘主要被以下3个"大户"占据：                          |
|                                                                    |
|     #1 微信聊天记录 (12.3 GB)  -- 建议迁移到D盘                     |
|     #2 VS Code 扩展及缓存 (4.7 GB) -- 部分可清理                   |
|     #3 Windows 临时文件 (3.2 GB)  -- 安全，可直接清理                |
|                                                                    |
|  [AI 一键执行全部建议]  [查看详细分析]                               |
|                                                                    |
+--------------------------------------------------------------------+
```

**AI 摘要生成逻辑：**
- 启动后后台自动完成全盘扫描
- 将扫描结果发送给 AI，要求生成一段精要摘要（限 150 字以内）
- AI 按占用大小排序，给出 TOP 3-5 个占用源
- 每条附带具体操作建议（迁移 / 清理 / 忽略）
- 提供「AI 一键执行」按钮，AI 按顺序自动执行所有建议的操作

#### 1.3 智能建议面板（自动分析，无需用户操作）

系统启动后自动后台扫描，生成以下建议（以 `SuggestionCard` 列表展示）：

| 建议类型 | 展示方式 (Widget) | 操作按钮 | AI 增强 |
|----------|-------------------|----------|---------|
| 安全清理建议 | `Card` 列表，每项含大小 `Text` | `ElevatedButton("一键执行")` | AI 自动标注优先级 |
| 迁移建议 | 高亮 `Card` 显示可迁移的大目录 | `ElevatedButton("一键迁移")` | AI 预估迁移收益 |
| 异常告警 | C盘 <10% 时顶部 `Banner` 组件横幅提醒 | `TextButton("去清理")` | AI 分析告警原因 |
| 大文件发现 | TOP 5 大文件 `Card` 卡片 | `IconButton`(打开位置) / `TextButton`(迁移) / `TextButton`(删除) | AI 判断是否可安全删除 |
| 占用者发现 | TOP 占用者排行卡片 | `TextButton("查看详情 -> 占用侦探")` | 链接到占用侦探模块 |

### 模块2：快速清理（全自动安全清理）

**目标：零风险、零配置、点一下就完成**

#### 2.1 自动化策略
- 启动时**自动扫描**所有安全清理项（无需手动触发，后台 `Isolate` 运行）
- 扫描完成后**自动计算**可释放空间总量
- 默认**全部勾选**安全项（`RiskLevel.safe` 级别），用户只需确认或取消个别项
- 提供「智能推荐模式」`Switch`：只勾选系统推荐的项（排除用户可能需要的）
- **AI 增强**：AI 根据用户的使用习惯，智能推荐哪些清理项应该保留（如：用户每天用的浏览器缓存可以保留最近7天的）

#### 2.2 清理项清单

| 清理项 | 路径/来源 | 风险等级 | 自动化程度 | AI 增强 |
|--------|-----------|----------|------------|---------|
| Windows 临时文件 | `%TEMP%`, `Windows\Temp` | `RiskLevel.safe` | 全自动 | AI 判断哪些进程正在使用 |
| 回收站 | `$Recycle.Bin` | `RiskLevel.safe` | 全自动 | AI 显示删除文件清单预览 |
| Windows 更新缓存 | `SoftwareDistribution\Download` | `RiskLevel.safe` | 全自动 | AI 提示是否正在更新中 |
| 缩略图缓存 | `thumbcache_*.db` | `RiskLevel.safe` | 全自动 | AI 提示重建所需时间 |
| 系统日志 | `Windows\Logs` | `RiskLevel.safe` | 全自动 | AI 过滤出关键日志不删 |
| 浏览器缓存 | Chrome/Edge/Firefox | `RiskLevel.safe` | 全自动（需关闭浏览器） | AI 智能保留近期缓存 |
| Prefetch 文件 | `Windows\Prefetch` | `RiskLevel.safe` | 全自动 | AI 识别常用程序保留预读 |
| 旧 Windows 安装 | `Windows.old` | `RiskLevel.warning` | 半自动（需二次确认） | AI 强烈警告不可恢复 |
| 错误报告 | `ProgramData\Microsoft\Windows\WER` | `RiskLevel.safe` | 全自动 | AI 关联错误类型说明 |

#### 2.3 清理流程（极简版 + AI 守护）

```
打开应用 -> 仪表盘已显示扫描结果(AI已标注优先级) -> 点击「一键清理」ElevatedButton
-> 弹出 AlertDialog 确认（AI 生成自然语言确认文案）-> 完成！-> AI 生成清理报告
```

- 确认弹窗 `AlertDialog` 内容（AI 生成）：
  ```
  AI 确认报告：
  即将为你释放 4.6 GB 空间，包括：
  [CheckIcon] Windows 临时文件 -- 2.3 GB （安全，无程序占用）
  [CheckIcon] 浏览器缓存 -- 1.8 GB （已保留近7天常用缓存）
  [CheckIcon] 回收站 -- 0.5 GB （共 23 个文件）
  
  AI 建议：这些都可以放心清理，不会影响你的正常使用。
  [取消 TextButton]                    [确认清理 ElevatedButton]
  ```

- 清理完成后弹出 AI 报告：
  ```
  [AI Icon] 清理完成！
  已释放 4.6 GB 空间，C盘使用率从 82% 降至 76%。
  下一步建议：微信占用了 12.3 GB，是否要迁移到D盘？
  [稍后再说 TextButton]  [去迁移 ElevatedButton]
  ```

### 模块3：占用侦探（新增）— C盘空间"嫌疑人"排查系统

**核心差异化功能 — 让用户知道"到底谁在偷吃我的C盘"**

#### 3.1 设计理念

> **传统清理工具只告诉你"有哪些垃圾"，CleanC 占用侦探告诉你"是谁在占用"。**
>
> **不是列一堆文件路径让用户自己猜，而是直接给出"罪魁祸首"排行榜。**

#### 3.2 检测维度（多维度画像）

| 维度 | 检测内容 | 排名依据 | 展示组件 |
|------|----------|----------|----------|
| **软件维度** | 每个软件在C盘的总占用（安装目录 + 数据目录 + 缓存目录） | 总占用大小降序 | `AppRankCard` 列表 |
| **文件夹维度** | C盘下一级/二级文件夹的大小排名 | 文件夹大小降序 | `FolderRankCard` 列表 |
| **文件类型维度** | 按扩展名归类统计（.log/.tmp/.cache/.mp4 等） | 同类总大小降序 | `FileTypePieChart` 饼图 |
| **时间维度** | 最近7天/30天内新增/增长的文件 | 增长量降序 | `TimelineChart` 时间轴 |
| **隐藏维度** | 隐藏文件/系统保护区域内的异常占用 | 异常程度排序 | `HiddenOccupancyCard`（需管理员权限） |

### 模块4：深度扫描

**目标：发现深层冗余，但保持用户控制权**

#### 4.1 大文件扫描
- 自动扫描 C 盘 >50MB 的文件（阈值可通过 `Slider` 调整）
- 按大小排序，`ListView` 展示：文件名 / 路径 / 大小 `Chip` / 最后访问时间
- **AI 增强**：每行附加 AI 判断标签 — "这是 XX 软件的缓存文件，可以安全删除"
- 每行操作 `ButtonBar`：「打开位置」「迁移到其他盘」「加入忽略名单」「问 AI」

#### 4.2 重复文件检测
- 基于 SHA-256 哈希值检测完全相同的文件
- 分组展示（`ExpansionTile`）：保留一份，其余标记为可删除
- **AI 增强**：AI 自动判断应该保留哪一份（如：保留修改时间最新的、路径最短的）
- 用户确认后才执行删除（`AlertDialog` 二次确认）

#### 4.3 卸载残留扫描
- 对比已安装程序列表与残留注册表项/文件夹
- 标记可疑残留（`RiskBadge.warning`），用户决定是否清理
- **AI 增强**：AI 分析残留内容，判断是否真的无用（如：某些残留可能是用户数据）

### 模块5：软件迁移（全自动向导式 + AI 辅助决策）

**核心差异化功能 -- 把软件从C盘搬走，而非删掉**

#### 5.1 自动化设计原则

> **用户只需做一件事：选择要迁移的软件。其余全部自动完成。AI 帮你做最优决策。**

#### 5.2 迁移流程（全自动向导，`Stepper` 组件 + AI 建议）

```
步骤1: 自动扫描（后台 Isolate 运行，无需等待）
   |-> 自动列出所有安装在C盘的软件（ListView 展示）
   |-> 自动标注每个软件的迁移可行性（CompatibilityBadge 组件）
   |-> AI 为每个软件生成迁移风险评估和建议
   
步骤2: 用户选择（唯一需要动手的环节）
   |-> Checkbox 勾选想要迁移的软件（支持多选）
   |-> AI 高亮推荐优先迁移的软件（"这个最占地方且最安全迁移"）
   
步骤3: 自动规划（全自动 + AI 优化）
   |-> AI 选择最佳目标盘（综合考虑空间、速度、已有同类软件分布）
   |-> 自动计算所需空间与目标盘剩余空间
   |-> 显示迁移预览 MigratePreviewCard（源路径 -> 目标路径）
   |-> AI 给出迁移顺序建议（先迁什么后迁什么避免冲突）
   
步骤4: 用户确认（一次点击）
   |-> 展示迁移摘要 AlertDialog + AI 风险评估（自然语言）
   |-> 点击「开始迁移」ElevatedButton
   
步骤5: 全自动执行（全程 LinearProgressIndicator 进度展示）
   |-> 自动创建还原点
   |-> 移动文件
   |-> 创建符号链接（如需要）
   |-> 更新注册表/快捷方式
   |-> 自动验证软件可正常启动
   
步骤6: 迁移完成
   |-> 展示结果报告 ResultDialog（成功/失败/注意事项 CheckIcon/CrossIcon）
   |-> AI 生成迁移后建议（"VS Code 迁移后首次启动可能略慢，属于正常现象"）
   |-> 提供一键回滚 ElevatedButton（30天内有效）
```

#### 5.3 迁移兼容性自动判断（`CompatibilityBadge` 组件 + AI 评估）

| 类型 | 判断逻辑 | 迁移方式 | CompatibilityBadge | AI 附加值 |
|------|----------|----------|---------------------|-----------|
| 绿色/便携软件 | 无注册表依赖 | 直接移动文件夹 | `compatible` (绿色) | AI 补充说明影响范围 |
| 安装型软件 | 有注册表项但路径集中 | 移动 + 符号链接 | `compatible` (绿色) | AI 列出所有受影响的注册表项 |
| 系统服务依赖 | 注册表散落多处 | 不建议迁移 | `incompatible` (灰色禁用) | AI 解释为什么不建议 |
| UWP/Microsoft Store 应用 | 封装在 AppX 内 | 不支持迁移 | `incompatible` (灰色禁用) | AI 说明替代方案 |

#### 5.4 批量迁移
- 支持一次性 `Checkbox` 多选后批量迁移
- **AI 自动排序**迁移顺序（考虑依赖关系、空间利用率、冲突避免）
- 总进度条 `LinearProgressIndicator` + 单个软件子进度 `LinearProgressIndicator`
- 失败自动跳过并记录原因（`ErrorCard`），不影响其他软件迁移
- AI 对失败项给出诊断建议

#### 7.8 隐私与安全

| 措施 | 说明 |
|------|------|
| 文件名脱敏 | 发送给 AI 的文件路径自动脱敏用户名（`C:\Users\xxx\` -> `C:\Users\[用户]\`） |
| 本地优先 | 所有确定性操作由本地规则引擎完成，AI 仅负责决策辅助 |
| 数据最小化 | 仅发送必要的元数据（路径、大小、类型），不发送文件内容 |
| 用户知情 | AI 对话中明确标注哪些数据会被发送到云端 |
| 离线模式 | 支持纯本地模式（AI 功能降级为基础规则推荐） |
| 可配置 | 设置中可开关 AI 功能，也可选择 AI 服务商 |

### 模块8：空间监控

- C盘空间低于阈值时系统托盘弹窗提醒（`SystemTray` + `Notification`，默认10%，可配置 `Slider`）
- **AI 增强**：告警消息由 AI 生成自然语言描述，而非冷冰冰的"空间不足"
- 空间变化历史曲线图（7天/30天）（`LineChart` 组件）
- **AI 增强**：图表上的关键节点附 AI 注释（"这一天暴涨是因为安装了 Steam"）
- 可配置定时自动清理计划（每周/每天）（`ScheduleSetting` 页面）
- **AI 增强**：AI 帮用户选择最佳清理频率（"根据你的使用习惯，建议每周三自动清理一次"）
- 清理历史记录（每次清理了什么、释放了多少）（`HistoryTimeline` 组件）

## 五、安全机制

| 机制 | 说明 | UI 表现 | AI 参与 |
|------|------|---------|---------|
| 白名单机制 | 一键清理只清理白名单内的路径，绝不越界 | 设置页白名单管理 `ListView` | AI 维护动态白名单 |
| 黑名单保护 | `System32`、`WindowsApps`、驱动目录等永不触碰 | 内置不可更改 | AI 拦截危险操作请求 |
| 还原点 | 每次迁移前自动创建系统还原点 | 迁移时底部 `SnackBar` 提示"已创建还原点" | AI 记录还原点关联的操作 |
| 操作日志 | 记录所有文件操作，支持查看和回滚 | 设置页操作日志 `DataTable` | AI 可用自然语言解释日志 |
| 回收站中转 | 删除文件优先进回收站（可配置永久删除） | 设置页 `Switch` 切换 | AI 提醒永久删除的风险 |
| 权限最小化 | 仅在必要时请求管理员权限，明确告知用途 | 权限请求 `AlertDialog` 说明用途 | AI 用通俗语言解释为何需要权限 |
| 30天回滚 | 所有迁移操作30天内支持一键回滚 | 每条历史记录旁 `OutlinedButton("回滚")` | AI 评估回滚影响 |
| AI 安全护栏 | AI 的操作建议必须经过用户最终确认才执行 | 确认弹窗中 AI 用自然语言说明后果 | AI 自我约束不越权 |

#### A.3 文件锁定处理完全缺失（高危）

**现状问题：** 清理/迁移时如果目标文件被进程占用（如浏览器正在运行时清理缓存、微信正在运行时迁移聊天记录），会导致操作失败甚至数据损坏。

**补充设计：**

// 文件锁定检测服务
class FileLockService {
  /// 检测文件/文件夹是否被占用
  Future<LockStatus> checkLock(String path) async {
    // 使用 Win32 API: CreateFile with FILE_SHARE_NONE
    // 或调用 handle.exe / tasklist 解析
  }
  
  /// 获取锁定该文件的进程列表
  Future<List<ProcessInfo>> getLockingProcesses(String path) async {
    // 返回: 进程名 / PID / 可否安全关闭
  }
  
  /// 尝试安全关闭锁定进程（需用户确认）
  Future<bool> requestCloseProcess(int pid) async {
    // 1. 弹窗告知用户哪个程序需要关闭
    // 2. 用户确认后发送 WM_CLOSE（非强制杀死）
    // 3. 等待进程退出后重试操作
    // 4. 超时后提示用户手动关闭
  }

  /// 清理时的锁定处理策略
  LockStrategy determineStrategy(LockStatus status) {
    switch (status) {
      case LockStatus.notLocked:
        return LockStrategy.proceed;           // 直接执行
      case LockStatus.lockedBySafeProcess:
        return LockStrategy.closeAndRetry;     // 安全关闭后重试
      case LockStatus.lockedBySystemProcess:
        return LockStrategy.skip;              // 跳过此项
      case LockStatus.lockedByUnknownProcess:
        return LockStrategy.promptUser;        // 让用户决定
    }
  }
}
```

**UI 表现：** 扫描结果中每个被锁定的项目显示 `LockedBadge`（灰色锁图标），点击可查看是哪个进程占用。

#### B.2 SSD 与 HDD 差异化策略缺失

**现状问题：** 对 SSD 和 HDD 用户使用相同的策略，但两者优化方向完全不同。

**补充设计：**

```
磁盘类型检测与差异化策略:
+-- 启动时自动检测 C 盘类型 ----------------------------+
|  使用 Win32 IOCTL_DISK_GET_DRIVE_PROPERTIES 检测      |
|  结果: SSD / HDD / SSHD(混合) / NVMe                   |
+-----------------------------------------------------+

SSD 用户专属建议:
  - 不推荐碎片整理（伤硬盘），改为 TRIM 优化建议
  - 迁移优先于删除（减少写操作延长寿命）
  - 禁用休眠文件释放空间（SSD重启快不需要休眠）
  - 推荐关闭 Superfetch/Sysmain 服务
  - 页面文件建议固定大小避免频繁伸缩

HDD 用户专属建议:
  - 可推荐定期碎片整理
  - 临时文件可以更激进地清理
  - 大文件迁移效果更明显（HDD寻道慢）
  - Prefetch 预读保留价值更高

UI表现:
  仪表盘显示磁盘类型标签: [SSD] 或 [HDD]
  AI 建议根据磁盘类型调整推荐策略
```

### E. 分发与信任

#### E.1 代码签名与数字证书

**现状问题：** 下载的 .exe 文件如果没有数字签名，Windows SmartScreen 会弹出"Windows 保护你的电脑"的吓人警告，很多用户不敢运行。

**补充设计：**

```
代码签名方案 (考虑成本与公益定位):
方案A (推荐初期): EV代码签名证书
  - 成本: 约 $200-400/年 (Comodo/Sectigo 等)
  - 效果: 立即建立信任，SmartScreen 信誉快速累积
  - 适合: 正式发布版本
  
方案B (免费替代): Authenticode 普通签名
  - 成本: 约 $70-100/年
  - 效果: 有签名但 SmartScreen 仍可能警告（需用户"仍要运行"）
  - 适合: 开发测试阶段 / 预算有限时

方案C (零成本): 无签名 + 发布哈希值
  - 在 GitHub Release 页面公布 exe 的 SHA256 哈希值
  - 用户可自行验证文件完整性
  - 配合详细的安装说明降低用户顾虑
  - 适合: 早期开发 / 内部测试

分发时附加:
  - 每个版本的 SHA256 校验值
  - 病毒扫描报告（VirusTotal 链接，应显示全绿）
  - "为什么可以信任 CleanC" 说明页面
```

#### E.2 便携版配置随身策略细节

**补充设计：**

```
便携版目录结构:
CleanC_Portable\
  ├── CleanC.exe              # 主程序
  ├── CleanC.dll              # 依赖库
  ├── data\                   # 运行时数据（随便携版带走）
  │   ├── cleanc.db           # SQLite数据库（操作日志/回滚/设置）
  │   ├── scans\              # 扫描结果缓存

  ├── config\                 # 用户配置（随便携版带走）
  │   ├── settings.json       # 用户偏好设置
  │   ├── ai_config.json      # AI配置（API Key等）
  │   └── clean_rules.json    # 自定义清理规则
  └── logs\                   # 日志文件

关键行为:
  - 首次运行自动在同目录创建 data/config/logs
  - 配置和数据永远存储在程序目录（非 AppData）
  - U盘插入任意电脑即可保持个人配置和历史
  - 多电脑场景: 每台电脑的扫描数据独立存储（按主机名分子目录）

#### E.3 命令行接口 (CLI) 模式

**为高级用户和自动化场景提供命令行支持：**

```
CLI 用法示例:
# 基本扫描
CleanC.exe scan --drive C

# 执行安全清理（非交互模式，适合定时任务）
CleanC.exe clean --safe --yes

# 完整清理+迁移（使用AI规划）
CleanC.exe full-auto --target-drive D

设计原则:
- 所有破坏性操作默认需要 --yes 确认参数
- 输出同时写到控制台和日志文件
- 支持 JSON 输出格式方便程序化处理
- 退出码遵循 Unix惯例: 0=成功, 1=通用错误, 2=参数错误
```

### F. 边界情况与鲁棒性

#### F.1 迁移过程中目标盘空间不足

**现状问题：** 迁移前检查了空间，但迁移过程中目标盘可能被其他程序写入导致空间不足。

**补充设计：**

```
目标盘空间动态监控:
+-- 迁移开始前 ----------------------------------------+
|  要求: 目标盘空闲 >= 源大小 * 1.2 （20%安全余量）       |
+-----------------------------------------------------+
+-- 迁移过程中 ----------------------------------------+
|  每复制完一批文件（如100个）后重新检查目标盘空间        |
|  如余量 < 源剩余大小的 1.1倍:                          |
|    -> 暂停迁移                                         |
|    -> 弹出警告: "目标盘空间不足，还剩 X GB 需要写入，    |
|       但目标盘仅剩 Y GB。请释放一些空间后点击继续。"      |
|    -> 提供「更换目标盘」选项                            |
|    -> 提供「放弃本次迁移」（已复制的文件保留，不删除）    |
+-----------------------------------------------------+
```

#### F.2 多用户 Windows 环境支持

**现状问题：** Windows 支持多用户，每个用户的 AppData 不同。CleanC 需要明确：

**补充设计：**

```
多用户支持策略:
+-- 当前用户模式 (默认) --------------------------------+
|  - 只操作当前登录用户的用户目录                         |
|  - 只修改当前用户的注册表 (HKCU)                       |
|  - 设置和日志存储在当前用户的 AppData 下               |
|  - 这是99%用户的使用场景                                |
+-----------------------------------------------------+
+-- 管理员全局模式 (可选) -------------------------------+
|  - 需要显式切换（设置页开关）                           |
|  - 可扫描所有用户目录                                  |
|  - 可清理公共临时文件 (%WINDIR%\Temp 等)                |
|  - 修改系统级注册表 (HKLM)                             |
|  - UI 上明确标注当前模式: "[当前用户]" / "[管理员全局]"  |
|  - 切换到管理员模式时需要二次确认                       |
+-----------------------------------------------------+
```

#### F.3 特殊路径与长路径处理

**现状问题：** Windows 最大路径长度限制（MAX_PATH = 260 字符），某些深层嵌套路径（如 node_modules、Python venv）会超出限制。

**补充设计：**

```
长路径处理策略:
+-- 启用长路径支持 ------------------------------------+
|  在 app manifest 中声明 longPathAware: true            |
|  使用 \\?\ 前缀绕过 260 字符限制                        |
|  Win32 API 调用统一使用 Unicode 版本 (W 后缀)          |
+-----------------------------------------------------+
+-- 特殊路径处理 --------------------------------------+
|  网络路径 (\\server\share): 跳过，提示不支持           |
|  虚拟路径 (shell:Desktop): 转换为真实路径后再操作       |
|  符号链接/联接点: 递归时跟踪链接（可选配置不跟踪）       |
|  NTFS 流 (Alternate Data Streams): 保留，不删除        |
|  保留字设备名 (CON/AUX/PRN 等): 自动跳过               |
+-----------------------------------------------------+
```

#### F.4 AI 免费额度耗尽时的降级体验

**现状问题：** 免费 API 有速率限制，高频使用可能触发限制。

**补充设计：**

```
AI 额度管理与降级:
+-- 额度监控 ------------------------------------------+
|  维护本地计数器: 今日已用 token 数 / 请求数             |
|  接近限额时提前切换到节省模式                          |
+-----------------------------------------------------+
+-- 降级层级 ------------------------------------------+
|  Level 0 (完整AI): 云端LLM + Function Calling          |
|  Level 1 (节省模式): 减少AI调用频率，合并多条建议       |
|  Level 2 (混合模式): 仅关键操作使用AI，其余用本地规则   |
|  Level 3 (离线模式): 完全使用本地规则引擎               |
+-----------------------------------------------------+
+-- 用户告知 ------------------------------------------+
|  切换级别时 discreetly 通知:                           |
|  "AI 已切换到智能建议模式（今日额度将尽），              |
|   核心功能不受影响。"                                  |
|  设置页显示当前级别和剩余额度的估算                     |
|  提供"配置自定义API Key"选项以突破免费限制              |
+-----------------------------------------------------+
```
